# Product Requirements Analyst Plugin

製品アイデアを包括的な要件ドキュメントに変換し、技術的実現可能性を検証するClaude Codeエージェントプラグインです。

## 概要

このプラグインは、製品要件分析の専門家として、ユーザーのアイデアを構造化された実行可能な要件に変換します。新機能や新規アプリケーションの企画段階で、潜在的なリスクやギャップを特定しながら、包括的なドキュメントを作成します。

## 主な機能

- **要件定義の構造化**: ユーザーアイデアを体系的な要件ドキュメントに変換
- **技術的実現可能性分析**: 技術スタック、API制限、技術的制約の検討
- **リスク評価**: 見落としがちなリスクやギャップを特定
- **アーキテクチャ設計**: システムアーキテクチャと設計仕様の提案
- **競合分析**: 市場の競合状況と差別化要因の評価

## 使用方法

### インストール

```bash
# マーケットプレイスを追加
/plugin marketplace add ariela/claude-plugins

# プラグインをインストール
/plugin install product-requirements-analyst@ariela-plugins
```

### エージェントの呼び出し

このプラグインは、以下のようなシナリオで自動的に呼び出されます:

#### 新規アプリアイデアの分析

```
ユーザー: ペットオーナー向けのSNSアプリを作りたい
Claude: 製品要件分析エージェントを使って、このアイデアを分析し、包括的な要件ドキュメントを作成します。
```

#### 新機能の技術的実現可能性検証

```
ユーザー: 既存プラットフォームにリアルタイムビデオチャット機能を追加できますか？
Claude: 製品要件分析エージェントを使って、技術的実現可能性を分析し、要件を作成します。
```

## 分析フレームワーク

このエージェントは、以下の観点から体系的に評価を行います:

1. **問題定義とターゲット市場の検証**
2. **技術アーキテクチャと実現可能性の制約**
3. **リソース要件とタイムラインへの影響**
4. **リスク評価と緩和戦略**
5. **成功指標と検証基準**
6. **競合状況と差別化要因**

## 出力

要件ドキュメントは `docs/<feature_name>/requirements.md` に作成されます。必要に応じて、補足ファイルも同じディレクトリに配置されます。

## コミュニケーションの特徴

- **無駄のない直接的な表現**: 社交辞令を排し、本質的な内容から開始
- **建設的な批判**: 論理的エラーやリスクを客観的に指摘し、事実に基づく代替案を提示
- **誠実な分析**: イエスマンにならず、正直で批判的な分析を提供

## 技術仕様

- **モデル**: Claude Sonnet
- **思考言語**: 英語（論理的精度を確保し、幻覚を減少）
- **出力言語**: 日本語
- **スコープ**: 要件定義とアーキテクチャ設計（実装コードは含まない）
